# 修复跳转按钮无响应问题

## 问题
用户反馈"打开小红书"按钮点击后没有反应。

## 原因分析
之前的实现使用了 iframe 方式尝试多种 scheme，逻辑过于复杂，可能导致：
1. iframe 方式在某些浏览器被拦截
2. 异步检测逻辑不可靠
3. scheme 尝试顺序可能有问题

## 修复方案
简化跳转逻辑：
1. 点击按钮直接尝试跳转（不通过 iframe）
2. iOS: 直接使用 `window.location.href = 'xhsdiscover://'`
3. Android: 使用 intent 方式
4. 2秒后如果没有跳转成功，自动 fallback 到真实链接
5. 添加按钮状态反馈（显示"正在打开..."）

## 代码变更
```javascript
function openApp() {
    // 显示加载提示
    const btn = document.getElementById('btnOpen');
    btn.innerHTML = '⏳ 正在打开...';
    btn.disabled = true;
    
    if (isWechat()) {
        // 微信内提示
        document.getElementById('wechatTips').classList.add('active');
        btn.innerHTML = '📱 打开小红书';
        btn.disabled = false;
        return;
    }
    
    // 直接跳转
    if (isIOS()) {
        window.location.href = 'xhsdiscover://';
    } else {
        window.location.href = 'intent://xiaohongshu.com#Intent;package=com.xingin.xhs;scheme=xhs;end;';
    }
    
    // 2秒后 fallback
    setTimeout(() => {
        window.location.href = realUrl;
    }, 2000);
}
```
